# Methodology

## 基本原则

1. 关键结论尽量由两个独立来源交叉验证。
2. 明确区分实际值、预测值和分析者估算。
3. 所有金额注明币种、名义或实际口径以及汇率日期。
4. 所有市场规模注明地域、产品边界和统计时间。
5. 对无法验证的数据明确标注不确定性。

## 市场规模

优先同时使用自上而下和自下而上两种方法，并解释差异：

- 自上而下：从宏观市场或权威统计逐层拆分。
- 自下而上：从客户数、销量、价格或公司收入汇总。

## 来源分级

| 等级 | 来源类型 | 使用建议 |
| --- | --- | --- |
| A | 政府统计、监管文件、审计披露 | 可作为核心事实依据 |
| B | 行业协会、公司材料、可信数据库 | 需要检查定义和利益关系 |
| C | 新闻、专家观点、二手报告 | 用于线索并尽量交叉验证 |

## 可复现性

数据清洗逻辑应放入 `src/`，探索过程可放入 `notebooks/`。处理后的数据需在 `data/README.md` 记录输入、转换步骤和输出。
